ANALYSIS RULES:
1. MODE ADAPTATION:
   - "joined": Joined DF is the primary truth for relationships (e.g., accounts + transactions).
   - "both": Start portfolio-level (joined), then drill into account/instrument nuances.
   - "separate": Identify divergence patterns across independent financial entities.

2. DATA INTEGRITY:
   - NO IDs: Never reference account IDs, transaction IDs as metrics.
   - DISCRETE: Treat transaction type, risk band, and tier codes as categorical.
   - SIGNIFICANCE: Correlations < |0.10| = "Noise".
   - QUALITY: Apply SEMANTIC NULLS PROTOCOL (rule 6) — not all nulls are data failures.
   - CAUSATION: Use descriptive language unless correlation > 0.6.

3. DATA HONESTY: No speculation. State "inconclusive" when evidence is insufficient.
   For mandatory field nulls >5% (e.g., Balance, Transaction Date), you MUST add a "Data Limitations" subsection.
   For sparse event columns (e.g., Default, Fraud Flag), analyze the non-null "Risk Cohort" — do NOT add a limitations subsection.

4. STRUCTURE:
   - Link transaction behavior to account/portfolio characteristics.
   - Directly address CROSS-SHEET HYPOTHESIS in conclusion.
   - Flag any data point that contradicts the hypothesis.

5. OUTPUT FORMAT:
   - Use professional Markdown.
   - Every claim MUST cite a specific metric from the JSON.

6. SEMANTIC NULLS PROTOCOL — Apply BEFORE any null-rate decision:
   CRITICAL: Do not treat all nulls as data failures. Classify every high-null column:
   a. TECHNICAL GAPS: Missing mandatory fields (e.g., Account Balance, Transaction Date, Customer ID). If >5% null → report as a Data Limitation.
   b. MEANINGFUL SPARSITY: Columns representing rare financial events (e.g., Default, Delinquency, Fraud Flag, Claim, Write-off). Nulls = valid "No Event" state.
   Rule: For sparse event columns, treat null as baseline (event did not occur). Analyze non-null rows as the "Risk Cohort" or "Default Cohort". Do NOT add a "Data Limitations" subsection for these columns.

GOLD STANDARD FINANCE FRAMEWORKS — Apply ALL three + two finance-specific lenses:

A. PARETO CONCENTRATION (80/20 Rule):
   - Which 20% of accounts, clients, or instruments drive 80% of AUM, revenue, or risk exposure?
   - Quantify exactly. Identify concentration risk (single entity >20% of total exposure).
   - Flag systemic risk if top-5 entities hold >50% of value.

B. BIMODAL DISTRIBUTION — Risk-Tier Separation:
   - Are transaction amounts or portfolio values bimodal? (Retail vs. Institutional tiers?)
   - Is risk score distribution bimodal? (Healthy vs. Distressed cohorts?)
   - Report both cluster means, volume share, and value share. Never blend tiers into one average.

C. PARADOX DETECTION — Counter-Intuitive Financial Findings:
   - Tag confirmed paradoxes with [PARADOX].
   - Classic finance paradoxes: high-risk assets with lowest returns; highest-fee products with best retention;
     accounts with most transactions generating least revenue; largest accounts with worst credit behavior.

D. FINANCE-SPECIFIC: Volatility & Concentration Risk:
   - Report coefficient of variation (std/mean) for key financial metrics as a volatility proxy.
   - Flag if top-3 entities account for >40% of a critical metric (concentration risk threshold).
   - Identify any zero-revenue or zero-activity accounts as dead-weight in the portfolio.

E. FINANCE-SPECIFIC: Fee & Margin Efficiency:
   - Correlate fee rates with transaction volume and net revenue.
   - Identify fee structures where volume does not compensate for operational cost.
   - Flag inverted relationships: high-volume, low-fee segments subsidized by low-volume, high-fee ones.
