ANALYSIS RULES:
1. MODE ADAPTATION:
   - "joined": Joined DF is the primary truth for relationships.
   - "both": Start macro (joined), then drill into per-sheet nuances.
   - "separate": Find patterns or contradictions across sheets.

2. DATA INTEGRITY:
   - NO IDs: Never use ID columns as metrics or for trends.
   - DISCRETE: Treat low-cardinality numeric columns (e.g., quantity, discount tiers) as categorical.
   - SIGNIFICANCE: Correlations < |0.10| = "Noise". Do not report as trends.
   - QUALITY: Apply SEMANTIC NULLS PROTOCOL (rule 6) — not all nulls are data failures.
   - CAUSATION: Use descriptive ("pattern") vs causal ("drove") unless correlation > 0.6.

3. DATA HONESTY: No speculation. If corr <0.1, state "inconclusive".
   For mandatory field nulls >5% (e.g., Price, Order Date, Customer ID), you MUST add a "Data Limitations" subsection.
   For sparse event columns (e.g., Return, Coupon), analyze the non-null "Target Segment" — do NOT add a limitations subsection.

4. STRUCTURE:
   - "Connected Insights": Link products to orders, customers to revenue, categories to margins.
   - Directly address CROSS-SHEET HYPOTHESIS in conclusion.
   - Explicitly flag data that contradicts the hypothesis.

5. OUTPUT FORMAT:
   - Use professional Markdown.
   - Every claim MUST cite a specific metric from the JSON.

6. SEMANTIC NULLS PROTOCOL — Apply BEFORE any null-rate decision:
   CRITICAL: Do not treat all nulls as data failures. Classify every high-null column:
   a. TECHNICAL GAPS: Missing mandatory fields (e.g., Price, Customer ID, Order Date). If >5% null → report as a Data Limitation.
   b. MEANINGFUL SPARSITY: Columns representing rare events (e.g., Return, Coupon Applied, Loyalty Tier Upgrade, Promotional Flag). Nulls = valid "No Event" state.
   Rule: For sparse event columns, treat null as baseline (event did not occur). Analyze non-null rows as the "Promotional Cohort" or "Target Segment". Do NOT add a "Data Limitations" subsection for these columns.

GOLD STANDARD RETAIL FRAMEWORKS — Apply ALL three + the two retail-specific lenses:

A. PARETO CONCENTRATION (80/20 Rule):
   - Which 20% of SKUs, categories, or customers generate 80% of revenue/profit?
   - Quantify precisely. Segment into Hero / Mid-Tier / Long-Tail.
   - Flag long-tail SKUs that consume inventory cost with negligible revenue contribution.

B. BIMODAL DISTRIBUTION — Dual-Segment Detection:
   - Price-point distributions: Is there a clear value vs. premium cluster?
   - Order-size distributions: Bulk buyers vs. single-unit shoppers?
   - NEVER average across segments — report each cluster's mean, volume share, and revenue share.

C. PARADOX DETECTION — Counter-Intuitive Retail Findings:
   - Tag confirmed paradoxes with [PARADOX].
   - Classic retail paradoxes to test: higher discount → lower total revenue (margin erosion);
     top-selling SKU has worst margin; highest-LTV customers order least frequently;
     premium category has highest return rate.

D. RETAIL-SPECIFIC: Discount Erosion Analysis:
   - Correlate discount depth with order volume AND margin simultaneously.
   - A discount that lifts volume but kills margin is a net negative — flag explicitly.
   - Identify the "discount sweet spot" tier where lift exceeds margin loss.

E. RETAIL-SPECIFIC: Category & Seasonal Concentration:
   - Which categories are over-indexed vs. the overall portfolio?
   - Identify any single-category dependency risk (>50% revenue in one category).
   - Flag seasonal spikes if date columns are present.
