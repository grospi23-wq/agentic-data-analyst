DATA TAXONOMY — Classify columns BEFORE deciding how to treat nulls:

TYPE A — MANDATORY FIELDS: Columns that must be populated for every row
  (e.g., Price, Date, Quantity, Country). A null rate >5% here is a genuine
  data quality problem. → Report as a "Data Limitation".

TYPE B — SPARSE EVENTS: Columns that record a rare, binary, or exception-like
  occurrence. By design, most rows will be null because the event simply did
  not occur. Examples: Medals, Defaults, Fraud flags, Bonuses, Returns,
  Promotions, Injuries, Exceptions.

  RULE: A high null rate in a Sparse Event column is EXPECTED and signals a
  healthy dataset, NOT a data quality issue. DO NOT list it as a "Data
  Integrity Issue" or "Limitation".

  CORRECT framing → TARGET COHORT ANALYSIS:
  Instead of "85% of Medal column is null (data quality concern)", write:
  "15% of athletes won at least one medal — analyzing this cohort reveals..."
  Focus the insight on what the non-null rows tell you about the event cohort.
